## Overview

The data layer uses Room Persistence Library to define entities that map to SQLite database tables. All models are located in `data/model/` and include entities for products, sales, purchases, categories, suppliers, and inventory movements.

## Core Entities

### SaleBrief

Simplified sale data with formatted date string.

**Location:** `data/model/SaleBrief.kt:6`

```kotlin
data class SaleBrief(
    val id: Long,
    val total: Double,
    val fecha: Long
) {
    val fechaString: String
        get() {
            if (fecha == 0L) return "-"
            val df = SimpleDateFormat("dd/MM/yyyy HH:mm", Locale.getDefault())
            return df.format(Date(fecha))
        }
}
```

## Database Relationships

### Foreign Key Constraints

- **Product → Category**: SET_NULL (allows category deletion)
- **SaleDetail → Sale**: CASCADE (deletes details when sale is deleted)
- **SaleDetail → Product**: RESTRICT (prevents deleting products with sales history)
- **PurchaseDetail → Purchase**: CASCADE (deletes details when purchase is deleted)
- **PurchaseDetail → Product**: RESTRICT (prevents deleting products with purchase history)
- **MovimientoInventario → Product**: CASCADE (deletes movements when product is deleted)

### Unique Constraints

- Product code must be unique
- Category name must be unique
- Supplier RUC must be unique
- Supplier name must be unique

## Best Practices
