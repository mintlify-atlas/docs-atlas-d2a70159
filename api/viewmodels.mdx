## Overview

Bodeguita uses **Android ViewModels** with **Kotlin Flows** for reactive state management following the MVVM (Model-View-ViewModel) architecture pattern. All ViewModels extend `ViewModel` and use coroutines for asynchronous operations.

## State Management Pattern

All ViewModels follow a consistent pattern:

- **StateFlow** for observable state exposed to the UI
- **SharedFlow** for one-time events (e.g., snackbar messages)
- **viewModelScope** for coroutine lifecycle management
- Repository pattern for data access

### Cart Operations

#### `addByCodeSafe(code: String, qty: Int)`

Adds a product to cart with validation.

```kotlin
fun addByCodeSafe(code: String, qty: Int) = viewModelScope.launch {
    if (code.isBlank()) {
        _events.tryEmit("Ingresa un código o selecciona un producto")
        return@launch
    }
    if (qty <= 0) {
        _events.tryEmit("Cantidad inválida (usa un número mayor a 0)")
        return@launch
    }
    try {
        addByCode(code.trim().uppercase(), qty)
        _events.tryEmit("Producto agregado")
    } catch (e: Exception) {
        _events.tryEmit("No se pudo agregar: ${e.message}")
    }
}
```

#### `changeQty(productId: Int, qty: Int)`

Updates quantity for an item in cart.

```kotlin
fun changeQty(productId: Int, qty: Int) {
    _cart.update { list ->
        list.map {
            if (it.product.id == productId)
                it.copy(qty = qty.coerceAtLeast(1))
            else it
        }
    }
}
```

#### `remove(productId: Int)`

Removes an item from the cart.

#### `clearCart()`

Empties the entire cart.

#### `finalizeSale()`

Completes the sale transaction.

```kotlin
fun finalizeSale() = viewModelScope.launch {
    try {
        val id = repo.makeSale(_cart.value)
        clearCart()
        _events.tryEmit("Venta realizada (#$id)")
    } catch (e: Exception) {
        _events.tryEmit(e.message ?: "Error al registrar la venta")
    }
}
```

### Key Methods

#### `addByCode(code: String, qty: Int, cost: Double)`

#### `addDraftLine(nameOrCode: String, qty: Int, cost: Double)`

#### `finalizeDraftPurchase(supplierId: Int)`

Materializes draft products and registers the purchase.

```kotlin
fun finalizeDraftPurchase(supplierId: Int) = viewModelScope.launch {
    val drafts = _draftCart.value
    if (drafts.isEmpty()) {
        _events.tryEmit("Agrega al menos un producto")
        return@launch
    }

    try {
        val items = mutableListOf<PurchaseRepository.CartItem>()

        for (d in drafts) {
            val code = d.nameOrCode.trim().uppercase()
            var p = productDao.findByCode(code)

            if (p == null) {
                // Create product on-the-fly
                productDao.insertIgnoreMinimal(
                    code = code,
                    name = d.nameOrCode.trim(),
                    price = d.cost,
                    stock = 0,
                    desc = ""
                )
                p = productDao.findByCode(code)
            }

            if (p != null) {
                items += PurchaseRepository.CartItem(
                    product = p,
                    qty = d.qty,
                    cost = d.cost
                )
            }
        }

        val id = repo.makePurchase(supplierId = supplierId, lines = items)
        clearDraftCart()
        _events.tryEmit("Compra registrada (#$id)")
    } catch (e: Exception) {
        _events.tryEmit(e.message ?: "Error al registrar compra")
    }
}
```

#### `generateReport(context: Context, onDone: (Uri?) -> Unit)`

Generates CSV report for sharing.

```kotlin
fun generateReport(context: Context, onDone: (Uri?) -> Unit) = viewModelScope.launch {
    try {
        val fileName = "reporte_cierre.csv"
        val file = File(context.cacheDir, fileName)

        val csvData = buildString {
            append("Total Ventas,${salesTotal.value}\n")
            append("Total Compras,${purchasesTotal.value}\n")
            append("Ganancia Neta,${netIncome.value}\n")
            append("Cantidad de Ventas,${salesCount.value}\n")
            append("Ticket Promedio,${avgTicket.value}\n")
        }

        FileOutputStream(file).use { it.write(csvData.toByteArray()) }

        val uri: Uri = FileProvider.getUriForFile(
            context,
            "${context.packageName}.fileprovider",
            file
        )

        onDone(uri)
    } catch (e: Exception) {
        onDone(null)
    }
}
```

fun addSupplier(ruc: String, nombre: String, telefono: String?) = viewModelScope.launch {
    val r = ruc.trim()
    val n = nombre.trim()
    val t = telefono?.trim().takeUnless { it.isNullOrBlank() }

    if (r.isBlank() || n.isBlank()) {
        _events.tryEmit("RUC y nombre son obligatorios")
        return@launch
    }

    val id = supplierDao.insertIgnore(Supplier(ruc = r, nombre = n, telefono = t))
    if (id == -1L) _events.tryEmit("El proveedor ya existe (RUC o nombre)")
    else _events.tryEmit("Proveedor agregado")
}

## ViewModelFactory Pattern

Bodeguita uses Factory pattern for ViewModel creation with repository dependencies.

### Example: ProductViewModelFactory

```kotlin
class ProductViewModelFactory(
    private val repository: ProductRepository
) : ViewModelProvider.Factory {
    override fun <T : ViewModel> create(modelClass: Class<T>): T {
        if (modelClass.isAssignableFrom(ProductViewModel::class.java)) {
            @Suppress("UNCHECKED_CAST")
            return ProductViewModel(repository) as T
        }
        throw IllegalArgumentException("Unknown ViewModel class")
    }
}
```

### Usage in Activity/Fragment

```kotlin
val productRepository = ProductRepository(database.productDao())
val productViewModel: ProductViewModel by viewModels {
    ProductViewModelFactory(productRepository)
}
```

**Similar factories exist for:**
- `SalesViewModelFactory`
- `PurchasesViewModelFactory`
- `DashboardViewModelFactory`
- `CashClosureViewModelFactory`
- `CategoryViewModelFactory`
- `SupplierViewModelFactory`

## Best Practices

<CardGroup cols={2}>
  <Card title="StateFlow for State" icon="database">
    Use `StateFlow` for observable state that the UI can collect and react to automatically.
  </Card>
  
  <Card title="SharedFlow for Events" icon="bolt">
    Use `SharedFlow` for one-time events like snackbar messages that shouldn't be replayed.
  </Card>
  
  <Card title="viewModelScope" icon="sync">
    Always launch coroutines in `viewModelScope` for automatic cancellation when ViewModel is cleared.
  </Card>
  
  <Card title="Validation" icon="shield-check">
    Perform input validation in ViewModel before repository calls to provide user feedback.
  </Card>
</CardGroup>
