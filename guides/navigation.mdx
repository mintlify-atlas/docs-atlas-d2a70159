Bodeguita uses Jetpack Compose Navigation with a modal drawer menu for navigating between screens. The navigation system is defined in `screens/MainNavigation.kt:71`.

## Navigation Architecture

The app uses a single-activity architecture with:

- **NavHost** - Manages navigation graph and screen transitions
- **NavController** - Controls navigation actions
- **ModalNavigationDrawer** - Side drawer menu for main navigation
- **Route-based navigation** - String-based route definitions

## NavHost Setup

The navigation graph is defined in `screens/MainNavigation.kt:187`:


### Routes with Parameters

For editing products, the app uses a parameterized route in `screens/MainNavigation.kt:205`:

```kotlin
composable(
    route = "product_registration/{productId}",
    arguments = listOf(
        navArgument("productId") { 
            type = NavType.StringType 
        }
    )
) { backStackEntry ->
    val productId = backStackEntry.arguments
        ?.getString("productId")
        ?.toIntOrNull()
    
    val products by productViewModel.products
        .collectAsState(initial = emptyList())
    val product = productId?.let { id -> 
        products.find { it.id == id } 
    }

    ProductRegistrationScreen(
        navController,
        productViewModel,
        categoryViewModel,
        product,
        onMenuClick = openDrawer
    )
}
```

### Drawer UI

```kotlin
val drawerState = rememberDrawerState(DrawerValue.Closed)
val scope = rememberCoroutineScope()
val navBackStackEntry by navController.currentBackStackEntryAsState()
val currentRoute = navBackStackEntry?.destination?.route

ModalNavigationDrawer(
    drawerState = drawerState,
    drawerContent = {
        ModalDrawerSheet {
            Spacer(Modifier.height(16.dp))
            drawerItems.forEach { item ->
                NavigationDrawerItem(
                    label = { Text(item.title) },
                    icon = { Icon(item.icon, contentDescription = item.title) },
                    selected = currentRoute == item.route,
                    onClick = {
                        scope.launch { drawerState.close() }
                        navController.navigate(item.route) { 
                            launchSingleTop = true 
                        }
                    },
                    modifier = Modifier.padding(
                        NavigationDrawerItemDefaults.ItemPadding
                    )
                )
            }
        }
    }
) {
    // NavHost content
}
```

## Navigation Patterns

### Opening the Drawer

```kotlin
val openDrawer: () -> Unit = { 
    scope.launch { drawerState.open() } 
}

// Pass to screens
DashboardScreen(
    navController = navController,
    onMenuClick = openDrawer
)
```

### Simple Navigation

```kotlin
// Navigate to a screen
navController.navigate("dashboard")

// Navigate with single top (prevent duplicates)
navController.navigate("products") {
    launchSingleTop = true
}
```

### Navigation with Parameters

```kotlin
// Navigate with a parameter
val productId = 123
navController.navigate("product_registration/$productId")
```

### Pop Back Stack

```kotlin
// Go back
navController.popBackStack()

// Go back to specific destination
navController.popBackStack("dashboard", inclusive = false)
```

### Clear Back Stack

```kotlin
// Navigate and clear back stack
navController.navigate("dashboard") {
    popUpTo("login") { inclusive = true }
    launchSingleTop = true
}
```

    ```kotlin
    composable(
        route = "edit_product?id={productId}",
        arguments = listOf(
            navArgument("productId") { 
                type = NavType.StringType
                nullable = true
                defaultValue = null
            }
        )

        val productId = backStackEntry.arguments
            ?.getString("productId")
            ?.toIntOrNull()
        
        // null = new product, non-null = edit existing
    ```

## Navigation Best Practices

<Card title="Use launchSingleTop" icon="layer-group">
  Prevent duplicate screens in the back stack by using `launchSingleTop = true` for main destinations.
</Card>

<Card title="Clear Back Stack on Logout" icon="arrow-right-from-bracket">
  When navigating to login, clear the entire back stack to prevent users from going back.
</Card>

<Card title="Handle Back Press" icon="arrow-left">
  For critical screens, handle back press to show confirmation dialogs:

  ```kotlin
  BackHandler(enabled = hasUnsavedChanges) {
      showDiscardDialog = true
  }
  ```
</Card>

<Card title="Pass ViewModels, Not Data" icon="database">
  Pass ViewModels to screens instead of individual data. ViewModels are scoped to the navigation graph and survive configuration changes.
</Card>

<Card title="Type-Safe Navigation" icon="shield-check">
  For complex apps, consider using type-safe navigation with the Navigation Compose library's type-safe APIs.
</Card>
