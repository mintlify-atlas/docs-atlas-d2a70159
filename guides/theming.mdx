Bodeguita uses Material Design 3 theming with full dark mode support. The theme system is built on Jetpack Compose and provides consistent styling across the app.

## Theme Architecture

The theme system consists of three main components located in `app/src/main/java/com/trabajo/minitienda/ui/theme/`:

- **Color.kt** - Color palette definitions
- **Type.kt** - Typography styles
- **Theme.kt** - Theme composition and dark mode handling

## Color Schemes

Bodeguita defines separate color schemes for light and dark modes in `ui/theme/Theme.kt:16`.

### Light Color Scheme

```kotlin
private val LightColorScheme = lightColorScheme(
    primary = PrimaryGreen,
    secondary = PrimaryGreenHover,
    background = BackgroundStart,
    surface = CardBackground,
    error = ErrorColor,
    onPrimary = CardBackground,
    onSecondary = PrimaryText,
    onBackground = PrimaryText,
    onSurface = PrimaryText,
    onError = CardBackground
)
```

### Dark Color Scheme

```kotlin
private val DarkColorScheme = darkColorScheme(
    primary = Color(0xFF66BB6A),           // Lighter green for dark mode
    secondary = Color(0xFF81C784),         // Secondary green
    background = Color(0xFF121212),        // Dark background
    surface = Color(0xFF1E1E1E),          // Dark cards
    error = Color(0xFFEF5350),            // Light red
```

## Dark Mode Management

Bodeguita uses `ThemeManager` located in `utils/ThemeManager.kt:10` to persist and manage theme preferences.

### ThemeManager Implementation

<Steps>
  <Step title="Initialize ThemeManager">
    ThemeManager uses SharedPreferences to persist the theme preference:

    ```kotlin
    class ThemeManager(context: Context) {
        private val prefs = context.getSharedPreferences(
            "mini_tienda_theme", 
            Context.MODE_PRIVATE
        )
        private val THEME_KEY = "is_dark_theme"
        
        private val _isDarkTheme = MutableStateFlow(getSavedTheme())
        val isDarkTheme: StateFlow<Boolean> = _isDarkTheme.asStateFlow()
    }
    ```
  </Step>

  <Step title="Toggle Theme">
    Use the `toggleTheme()` method to switch between light and dark modes:

    ```kotlin
    fun toggleTheme() {
        val newTheme = !_isDarkTheme.value
        _isDarkTheme.value = newTheme
        prefs.edit().putBoolean(THEME_KEY, newTheme).apply()
    }
    ```
  </Step>

  <Step title="Apply Theme in Composables">
    The theme is applied in `ui/theme/Theme.kt:44`:

    ```kotlin
    @Composable
    fun MiniTiendaTheme(
        darkTheme: Boolean = isSystemInDarkTheme(),
        content: @Composable () -> Unit
    ) {
        val colorScheme = if (darkTheme) DarkColorScheme else LightColorScheme
        val view = LocalView.current

        if (!view.isInEditMode) {
            SideEffect {
                val window = (view.context as Activity).window
                window.statusBarColor = colorScheme.primary.toArgb()
                WindowCompat.getInsetsController(window, view)
                    .isAppearanceLightStatusBars = !darkTheme
            }
        }

        MaterialTheme(
            colorScheme = colorScheme,
            typography = Typography,
            content = content
        )
    }
    ```
  </Step>
</Steps>

## Customizing Colors

<Steps>
  <Step title="Define Custom Colors">
    Add your custom colors to `ui/theme/Color.kt`:

    ```kotlin
    val CustomBlue = Color(0xFF2563EB)
    val CustomBlueDark = Color(0xFF1E40AF)
    ```
  </Step>

  <Step title="Update Color Schemes">
    Add your colors to the appropriate color scheme in `ui/theme/Theme.kt`:

    ```kotlin
    private val LightColorScheme = lightColorScheme(
        primary = CustomBlue,  // Changed from PrimaryGreen
        // ... other colors
    )
    ```
  </Step>

  <Step title="Use in Composables">
    Access theme colors using `MaterialTheme.colorScheme`:

    ```kotlin
    @Composable
    fun MyButton() {
        Button(
            colors = ButtonDefaults.buttonColors(
                containerColor = MaterialTheme.colorScheme.primary
            )
        ) {
            Text("Click me")
        }
    }
    ```
  </Step>
</Steps>

## Best Practices

<Card title="Always Use Theme Colors" icon="palette">
  Never hardcode colors. Always use `MaterialTheme.colorScheme` to ensure dark mode compatibility.
</Card>

<Card title="Test Both Themes" icon="sun-bright">
  Test all screens in both light and dark modes to ensure proper contrast and readability.
</Card>

<Card title="Semantic Color Names" icon="tag">
  Use semantic color roles (primary, secondary, error) instead of descriptive names (green, red) for better theme flexibility.
</Card>

<Card title="Consistent Typography" icon="font">
  Use predefined typography styles from `MaterialTheme.typography` for consistent text hierarchy.
</Card>
