Supplier Management enables store owners to maintain a database of suppliers (proveedores) with unique tax IDs, contact information, and automatic linking to purchases.

## Overview

Suppliers are critical for the purchase workflow in Bodeguita. Every purchase must be linked to a supplier, creating a complete audit trail of inventory sources.

### Key Features

- **Unique supplier identification** by RUC (tax ID) and name
- **Contact information** storage (phone numbers)
- **Quick creation** from purchase screen
- **Full CRUD operations** through dedicated management dialogs
- **Automatic linking** to purchase records
- **Reactive updates** across all screens

## Data Model

Suppliers are stored in Room with unique constraints:

```kotlin
// Supplier.kt
@Entity(
    tableName = "proveedor",
    indices = [
        Index(value = ["ruc"], unique = true),
        Index(value = ["nombre"], unique = true)
    ]
)
data class Supplier(
    @PrimaryKey(autoGenerate = true) val id: Int = 0,
    val ruc: String,        // Tax ID - must be unique
    val nombre: String,     // Name - must be unique
    val telefono: String? = null  // Optional phone number
)
```

<Warning>
  Both `ruc` (tax ID) and `nombre` (name) have **unique constraints**. Attempting to insert a duplicate will fail silently with `insertIgnore` returning `-1`.
</Warning>

## SupplierViewModel

The ViewModel provides reactive supplier management:

```kotlin
// SupplierViewModel.kt
class SupplierViewModel(
    private val supplierDao: SupplierDao
) : ViewModel() {

    // Reactive supplier list from Room
    val suppliers: StateFlow<List<Supplier>> =
        supplierDao.observeAll()
            .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), emptyList())

    // Event channel for UI feedback
    private val _events = MutableSharedFlow<String>(extraBufferCapacity = 1)
    val events: SharedFlow<String> = _events.asSharedFlow()

    /** Create supplier (if already exists by RUC or name, insertIgnore returns -1) */
    fun addSupplier(ruc: String, nombre: String, telefono: String?) = viewModelScope.launch {
        val r = ruc.trim()
        val n = nombre.trim()
        val t = telefono?.trim().takeUnless { it.isNullOrBlank() }

        if (r.isBlank() || n.isBlank()) {
            _events.tryEmit("RUC y nombre son obligatorios")
            return@launch
        }

        val id = supplierDao.insertIgnore(Supplier(ruc = r, nombre = n, telefono = t))
        if (id == -1L) _events.tryEmit("El proveedor ya existe (RUC o nombre)")
        else _events.tryEmit("Proveedor agregado")
    }

    /** Rename supplier by id */
    fun renameSupplier(id: Int, nuevoNombre: String) = viewModelScope.launch {
        val nuevo = nuevoNombre.trim()
        if (nuevo.isBlank()) return@launch
        supplierDao.rename(id, nuevo)
        _events.tryEmit("Proveedor actualizado")
    }

    /** Delete supplier */
    fun deleteSupplier(supplier: Supplier) = viewModelScope.launch {
        supplierDao.delete(supplier)
        _events.tryEmit("Proveedor eliminado")
    }
}
```

### StateFlow Pattern

```kotlin
// SupplierViewModel.kt:16
val suppliers: StateFlow<List<Supplier>> =
    supplierDao.observeAll()
        .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), emptyList())
```

<Info>
  The suppliers list is a **hot flow** that automatically updates all UI components when suppliers are added, renamed, or deleted. The `WhileSubscribed(5_000)` sharing strategy stops upstream Room queries 5 seconds after the last collector unsubscribes, optimizing resources.
</Info>

## Quick Create Dialog

Suppliers can be created on-the-fly from the Purchases screen:

```kotlin
// PurchasesScreen.kt:329
@Composable
private fun SupplierManagerDialog(
    onDismiss: () -> Unit,
    onSave: (ruc: String, nombre: String, telefono: String?) -> Unit
) {
    var ruc by remember { mutableStateOf("") }
    var nombre by remember { mutableStateOf("") }
    var telefono by remember { mutableStateOf("") }

    AlertDialog(
        onDismissRequest = onDismiss,
        title = { Text("Gestionar Proveedor") },
        text = {
            Column(verticalArrangement = Arrangement.spacedBy(12.dp)) {
                OutlinedTextField(
                    value = ruc,
                    onValueChange = { ruc = it },
                    label = { Text("RUC") }
                )
                OutlinedTextField(
                    value = nombre,
                    onValueChange = { nombre = it },
                    label = { Text("Nombre") }
                )
                OutlinedTextField(
                    value = telefono,
                    onValueChange = { telefono = it },
                    label = { Text("Teléfono") }
                )
            }
        },
        confirmButton = {
            TextButton(onClick = {
                onSave(ruc, nombre, telefono.ifBlank { null })
                onDismiss()
            }) { Text("Guardar") }
        },
        dismissButton = {
            TextButton(onClick = onDismiss) { Text("Cerrar") }
        }
    )
}
```

## Supplier Selection UI

Suppliers are selected using ExposedDropdownMenuBox:

```kotlin
// PurchasesScreen.kt:75
var selectedSupplierIdx by remember { mutableStateOf(-1) }
val selectedSupplier = suppliers.getOrNull(selectedSupplierIdx)

var expanded by remember { mutableStateOf(false) }

ExposedDropdownMenuBox(
    expanded = expanded,
    onExpandedChange = { expanded = !expanded }
) {
    OutlinedTextField(
        modifier = Modifier.menuAnchor().fillMaxWidth(),
        value = selectedSupplier?.nombre ?: "",
        onValueChange = {},
        readOnly = true,
        label = { Text("Selecciona un proveedor") },
        trailingIcon = {
            ExposedDropdownMenuDefaults.TrailingIcon(expanded = expanded)
        },
        singleLine = true
    )

    ExposedDropdownMenu(
        expanded = expanded,
        onDismissRequest = { expanded = false }
    ) {
        suppliers.forEachIndexed { index, s ->
            DropdownMenuItem(
                text = { Text("${s.nombre}  —  ${s.ruc}") },
                onClick = {
                    selectedSupplierIdx = index
                    expanded = false
                }
            )
        }
    }
}
```

<Info>
  The dropdown displays both supplier **name** and **RUC** for easy identification: "Distribuidora ABC — 20123456789"

## Best Practices

<Warning>
  **RUC Format**: The system doesn't validate RUC format. In production, you should validate that it matches the Peruvian RUC format (11 digits, specific prefixes, etc.).
</Warning>

<Note>
  **Duplicate Prevention**: The unique constraints on RUC and nombre prevent accidental duplicates. Always use `insertIgnore` and check for `-1` return value to handle conflicts gracefully.
</Note>

<Info>
  **Reactive Updates**: When a supplier is created in the quick dialog, all dropdowns and lists automatically update via StateFlow, so users see the new supplier immediately.
</Info>

### Recommended Workflow

<Steps>
  <Step title="Verify Supplier Existence">
    Before creating a purchase, check if the supplier exists in the dropdown
  </Step>

  <Step title="Create if Missing">
    Click "Gestionar" to open the quick create dialog
  </Step>

  <Step title="Enter Complete Information">
    Provide RUC, name, and phone (optional but recommended)
  </Step>

  <Step title="Save and Select">
    After saving, the new supplier appears in the dropdown automatically
  </Step>

  <Step title="Complete Purchase">
    Select the supplier and proceed with adding products
  </Step>
</Steps>

## SupplierDao Interface

The DAO provides database operations:

```kotlin
@Dao
interface SupplierDao {
    @Query("SELECT * FROM proveedor ORDER BY nombre ASC")
    fun observeAll(): Flow<List<Supplier>>

    @Insert(onConflict = OnConflictStrategy.IGNORE)
    suspend fun insertIgnore(supplier: Supplier): Long

    @Query("UPDATE proveedor SET nombre = :nuevoNombre WHERE id = :id")
    suspend fun rename(id: Int, nuevoNombre: String)

    @Delete
    suspend fun delete(supplier: Supplier)
}
```

<Info>
  The `observeAll()` query returns suppliers **alphabetically sorted by name** for consistent dropdown ordering.
</Info>
