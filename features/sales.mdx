The Sales Management module enables quick sale registration with shopping cart functionality, multi-product support, automatic inventory updates, and real-time price calculations including tax.

## Overview

SalesScreen provides a complete point-of-sale interface with:
- **Shopping cart** with add/remove functionality
- **Product lookup** by code or search
- **Multi-product sales** with quantity control
- **Automatic inventory deduction** on sale completion
- **Tax calculation** (18% IGV)
- **Responsive layout** for various screen sizes

## Data Model

Sales use a master-detail pattern with Room:

```kotlin
// Sale.kt
@Entity(tableName = "venta")
data class Sale(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val fecha: Long = System.currentTimeMillis(),
    val total: Double
)

@Entity(
    tableName = "detalle_venta",
    primaryKeys = ["sale_id", "product_id"],
    foreignKeys = [
        ForeignKey(
            entity = Sale::class,
            parentColumns = ["id"],
            childColumns = ["sale_id"],
            onDelete = ForeignKey.CASCADE
        ),
        ForeignKey(
            entity = Product::class,
            parentColumns = ["id"],
            childColumns = ["product_id"],
            onDelete = ForeignKey.RESTRICT
        )
    ]
)
data class SaleDetail(
    @ColumnInfo(name = "sale_id") val saleId: Long,
    @ColumnInfo(name = "product_id") val productId: Int,
    val cantidad: Int,
    @ColumnInfo(name = "precio_unit") val precioUnit: Double,
    val subtotal: Double
)
```

<Note>
  Sales use **CASCADE** delete: deleting a sale removes all its detail rows. Products use **RESTRICT**: you cannot delete a product that has been sold.
</Note>

    ```kotlin
    // SalesViewModel.kt:51
    fun addByCodeSafe(code: String, qty: Int) = viewModelScope.launch {
        // Validation
        if (code.isBlank()) {
            _events.tryEmit("Ingresa un código o selecciona un producto")
            return@launch
        }
        if (qty <= 0) {
            _events.tryEmit("Cantidad inválida (usa un número mayor a 0)")
            return@launch
        }
        try {
            addByCode(code.trim().uppercase(), qty)
            _events.tryEmit("Producto agregado")
        } catch (e: Exception) {
            _events.tryEmit("No se pudo agregar: ${e.message ?: "error desconocido"}")
        }
    }

    fun addByCode(code: String, qty: Int) = viewModelScope.launch {
        val p = productDao.findByCode(code.trim().uppercase())
        if (p == null) {
            _events.tryEmit("Código no encontrado")
            return@launch
        }
        if (qty <= 0) {
            _events.tryEmit("Cantidad inválida")
            return@launch
        }

        val current = _cart.value.toMutableList()
        val index = current.indexOfFirst { it.product.id == p.id }
        if (index >= 0) {
            // Update existing line
            val old = current[index]
            current[index] = old.copy(qty = old.qty + qty)
        } else {
            // Add new line
            current += CartItem(p, qty)
        }
        _cart.value = current
    }
    ```
  </Step>
</Steps>

<Info>
  If you add the same product multiple times, the quantities are **accumulated** in a single cart line rather than creating duplicate entries.
</Info>

## Finalizing the Sale

The sale is persisted with automatic inventory updates:

```kotlin
// SalesViewModel.kt:80
fun finalizeSale() = viewModelScope.launch {
    try {
        val id = repo.makeSale(_cart.value)
        clearCart()
        _events.tryEmit("Venta realizada (#$id)")
    } catch (e: Exception) {
        _events.tryEmit(e.message ?: "Error al registrar la venta")
    }
}
```

### Repository Transaction

The repository handles the complete transaction:

<Steps>
  <Step title="Insert Sale Master Record">
    Create the sale header with total and timestamp
  </Step>

  <Step title="Insert Sale Detail Lines">
    Add each cart item as a sale detail row
  </Step>

  <Step title="Update Product Stock">
    Deduct sold quantities from inventory
  </Step>

  <Step title="Record Inventory Movement">
    Log the stock change for audit trail
  </Step>
</Steps>

<Warning>
  All operations happen in a **database transaction**. If any step fails (e.g., insufficient stock), the entire sale is rolled back.
</Warning>

## Best Practices

<Info>
  **Search Optimization**: The product dropdown limits results to 12 items to prevent performance issues with large inventories. Users can type to filter further.
</Info>

<Note>
  **Quantity Accumulation**: Adding the same product twice combines quantities in one cart line, making it easy to see totals per product.
</Note>

<Warning>
  **Stock Validation**: The repository should validate stock levels before finalizing the sale. If stock is insufficient, the transaction fails and the cart remains intact.
</Warning>
