Cash Closure (Cierre de Caja) provides end-of-day financial summaries, calculating total sales, purchases, net profit, and generating exportable CSV reports for accounting.

## Overview

The Cash Closure screen helps store owners:
- **Track daily revenue** from sales
- **Monitor expenses** from purchases
- **Calculate net profit** (sales - purchases)
- **View key metrics** like average ticket and profit margin
- **Export reports** as CSV for accounting software
- **Reset daily data** to start fresh (with caution)

## Architecture

CashClosureViewModel aggregates data from multiple DAOs:

```kotlin
// CashClosureViewModel.kt:17
class CashClosureViewModel(
    private val saleDao: SaleDao,
    private val movementDao: MovementDao,
    private val purchaseDao: PurchaseDao,
) : ViewModel() {

    val salesTotal: StateFlow<Double> =
        saleDao.todaySalesTotal().stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), 0.0)

    val salesCount: StateFlow<Int> =
        saleDao.todaySalesCount().stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), 0)

    val purchasesTotal: StateFlow<Double> =
        purchaseDao.todayPurchasesTotal().stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), 0.0)

    val netIncome: StateFlow<Double> =
        combine(salesTotal, purchasesTotal) { s, p -> s - p }
            .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), 0.0)

    val avgTicket: StateFlow<Double> =
        combine(salesTotal, salesCount) { s, c -> if (c <= 0) 0.0 else s / c }
            .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), 0.0)

    val movementsToday: StateFlow<List<MovimientoInventario>> =
        movementDao.todayMovements()
            .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), emptyList())
}
```

## Calculated Metrics

### Average Ticket

Average sale amount per transaction:

```kotlin
// CashClosureViewModel.kt:36
val avgTicket: StateFlow<Double> =
    combine(salesTotal, salesCount) { s, c -> if (c <= 0) 0.0 else s / c }
        .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), 0.0)
```

<Info>
  **Formula**: `Average Ticket = Total Sales / Number of Transactions`
  
  If no sales occurred today, returns `0.0` to avoid division by zero.
</Info>

### Profit Margin

Percentage of sales revenue that is profit:

```kotlin
// CashClosureScreen.kt:303
private fun percent(net: Double, sales: Double): String =
    if (sales <= 0.0) "0.0% margen"
    else String.format(Locale.getDefault(),"%.1f%% margen", (net / sales) * 100.0)
```

<Info>
  **Formula**: `Profit Margin = (Net Income / Total Sales) × 100%`
  
  Example: If sales = S/ 1000 and net income = S/ 250, margin = 25.0%
</Info>

## CSV Report Generation

Export daily data to CSV for external accounting:

```kotlin
// CashClosureScreen.kt:132
Button(
    onClick = {
        vm.generateReport(context) { uri ->
            if (uri != null) {
                val intent = android.content.Intent().apply {
                    action = android.content.Intent.ACTION_SEND
                    type = "text/csv"
                    putExtra(android.content.Intent.EXTRA_STREAM, uri)
```

### ViewModel Report Logic

```kotlin
// CashClosureViewModel.kt:51
fun generateReport(context: Context, onDone: (Uri?) -> Unit) = viewModelScope.launch {
    try {
        val fileName = "reporte_cierre.csv"
        val file = File(context.cacheDir, fileName)

        val csvHeader = "Concepto,Valor\n"

        val csvData = buildString {
            append("Total Ventas,${salesTotal.value}\n")
            append("Total Compras,${purchasesTotal.value}\n")
            append("Ganancia Neta,${netIncome.value}\n")
            append("Cantidad de Ventas,${salesCount.value}\n")
            append("Ticket Promedio,${avgTicket.value}\n")
            append("Movimientos,${movementsToday.value}\n")
        }

        FileOutputStream(file).use { it.write((csvHeader + csvData).toByteArray()) }

        val uri: Uri = FileProvider.getUriForFile(
            context,
            "${context.packageName}.fileprovider",
            file
        )

        onDone(uri)
    } catch (e: Exception) {
        e.printStackTrace()
        onDone(null)
    }
}
```

<Steps>
  <Step title="Create CSV File">
    Generate `reporte_cierre.csv` in the app's cache directory
  </Step>

  <Step title="Write Data">
    Format metrics as CSV rows: "Concepto,Valor"
  </Step>

  <Step title="Generate URI">
    Use FileProvider to create a shareable content URI
  </Step>

  <Step title="Share Intent">
    Open Android share sheet for sending via email, cloud storage, etc.
  </Step>
</Steps>

## Day Reset Functionality

<Warning>
  **Destructive Operation**: Resetting the day **permanently deletes** today's sales, purchases, and inventory movements. This cannot be undone.
</Warning>

### ViewModel Reset Logic

```kotlin
// CashClosureViewModel.kt:44
fun resetDay(onDone: () -> Unit) = viewModelScope.launch {
    saleDao.deleteTodaySales()
    purchaseDao.deleteTodayPurchases()
    movementDao.deleteTodayMovements()
    onDone()
}
```

<Steps>
  <Step title="Delete Today's Sales">
    Remove all sale records created today
  </Step>

  <Step title="Delete Today's Purchases">
    Remove all purchase records created today
  </Step>

  <Step title="Delete Today's Movements">
    Remove all inventory movement logs created today
  </Step>

  <Step title="Callback">
    Trigger `onDone()` to hide loading spinner
  </Step>
</Steps>

<Note>

## Best Practices

<Info>
  **Daily Workflow**:
  1. Review metrics at end of day
  2. Generate CSV report
  3. Send report to accountant or cloud storage
  4. Only then reset day to start fresh
</Info>

<Warning>
  **Backup Strategy**: Implement automatic cloud backups before enabling day reset. Consider archiving data instead of deleting.
</Warning>

<Note>
  **Time-based Queries**: All "today" queries depend on accurate device time. Ensure the device clock is set correctly.
</Note>
